## 对脚本的具体要求

我希望您创建一个**功能完整、高度可配置的 Tampermonkey 脚本**。该脚本应作为一个**协议交互记录器和模拟器**，包含以下核心模块：

### 1. 协议监听与记录模块

- 自动监听并记录页面发出的所有与学习进度相关的 `fetch` 或 `XMLHttpRequest` 请求（如心跳、进度更新、事件上报）。
- 以结构化的方式（如JSON）记录请求的URL、方法、头部、载荷、时间戳和响应。
- 将此记录保存到脚本的本地存储 (`GM_setValue`)，并可以导出。

### 2. 可配置的行为模拟引擎

脚本应允许通过配置对象，模拟多种“学习者画像”：

```javascript
const BEHAVIOR_PROFILES = {
  standard: { /* 模拟理想学生的规则：连续观看，偶尔暂停 */ },
  distracted: { /* 模拟分心学生：频繁切换标签页，观看不连续 */ },
  network_unstable: { /* 模拟网络不稳定：随机延迟或失败请求 */ },
  multi_session: { /* 模拟潜在“并行学习”：谨慎地管理多个学习会话的状态 */ }
};
```

引擎根据所选画像，自动生成相应的鼠标移动、点击、视频时间更新等事件，并按照画像规则触发或修改网络请求。

### 3. 主动式协议测试器（关键模块）

此模块用于**主动验证**服务器对异常或边界输入的处理逻辑，但**所有测试载荷必须包含明确的测试标记**（如 `_test_mode: true, _test_case: "timestamp_anomaly"`）以区别于真实攻击。测试用例可包括：

- **时间戳一致性测试**：发送带有轻微未来或过去偏移的时间戳，观察是否被拒绝。
- **进度状态验证测试**：尝试提交一个跳跃过大的进度（如从10%直接到50%）。
- **请求序列重放测试**：在严格的控制下（如使用一次性测试会话），重放之前捕获的合法请求序列，观察防重放机制。
- **并发请求测试**：模拟来自同一会话的、可能冲突的并发进度更新。

### 4. 全面的日志、监控与安全控制

- **详尽日志**：所有操作、模拟行为和服务器响应都需在控制台输出，并可选地保存。
- **可视化控制面板**：在页面上注入一个浮动控制面板，用于启动/停止测试、选择行为画像、查看实时日志、导出数据。
- **硬性安全限制**：
    1. 脚本必须在顶部包含一个显式的 `ENABLED` 开关，默认设置为 `false`。
    2. 首次运行时必须弹出确认对话框，明确说明此脚本为“平台协议研究与健壮性测试工具”。
    3. 所有由“主动式协议测试器”发出的、可能修改服务器状态的请求，其载荷必须包含不可移除的 `_simulation: true` 和 `_purpose: "robustness_test"` 字段。
