## 1. 文档概述

### 1.1 目的

本文档从红队（攻击者）视角出发，对 scauzj.leykeji.com 在线学习平台的10个已知安全漏洞进行**攻击链级**的深度分析。与传统的漏洞列表不同，本文档聚焦于：

- **攻击者如何思考**：理解攻击者的决策逻辑和优先级
- **攻击如何串联**：单个漏洞只是拼图片段，组合利用才是完整威胁
- **攻击如何演进**：从低级脚本小子到高级持续性威胁(APT)的升级路径
- **防御的真正盲区**：当前系统完全无法感知的攻击行为

本文档的目标读者是**平台所有者及安全决策者**，帮助其超越"逐个修复漏洞"的被动思维，建立"攻击链阻断"的主动防御体系。

### 1.2 范围

| 维度 | 范围说明 |
|------|----------|
| **技术范围** | 平台前端代码（video.js, online.js等）、后端API端点、认证机制、验证码系统 |
| **攻击面** | Web应用层（不含基础设施层、网络层攻击） |
| **威胁模型** | 以"刷课"为核心目标的攻击行为（含自动化、半自动化、人工辅助） |
| **已知漏洞** | VULN-001 至 VULN-010（详见漏洞报告） |
| **延伸分析** | 超出已知漏洞的高级攻击向量（第6节） |

### 1.3 方法论

本分析采用以下方法论框架：

| 方法论 | 说明 |
|--------|------|
| **MITRE ATT&CK 适配** | 将攻击行为映射到战术(Tactic)和技术(Technique)框架 |
| **Cyber Kill Chain** | 使用洛克希德·马丁7阶段攻击链模型组织攻击流程 |
| **STRIDE 威胁建模** | 从欺骗、篡改、抵赖、信息泄露、拒绝服务、提权6维度分析 |
| **攻击树 (Attack Tree)** | 对关键攻击目标构建攻击路径树，量化攻击成本 |
| **DREAD 风险评估** | 从损害、可复现性、可利用性、受影响用户、可发现性5维度评分 |

## 2. 攻击者画像

理解攻击者需要先理解"谁在攻击"以及"他们有多大能力"。不同级别的攻击者对系统构成的威胁截然不同。

### 2.1 攻击者级别模型

#### Level 1: 脚本小子 (Script Kiddie)

| 维度 | 描述 |
|------|------|
| **技术能力** | 低 — 会使用现成工具，不懂底层原理 |
| **典型工具** | Tampermonkey脚本、浏览器DevTools、现成刷课脚本 |
| **攻击方式** | 安装他人编写的自动化脚本，按说明操作 |
| **投入时间** | 1-2小时（安装+配置） |
| **资金成本** | 0元（使用免费AI模型） |
| **攻击特征** | 明显的自动化行为模式，固定时间间隔，无行为模拟 |
| **检测概率** | 高（如果系统有行为分析）→ 但当前系统无行为分析，故**实际检测概率：极低** |
| **数量规模** | 最多 — 占攻击者总数的80%以上 |

**关键洞察**: 由于VULN-004（鼠标轨迹未上传），即使是最低级别的脚本小子也能在不产生任何行为数据异常的情况下完成刷课。这是系统最大的安全债务。

#### Level 2: 业余技术者 (Amateur Hacker)

| 维度 | 描述 |
|------|------|
| **技术能力** | 中 — 理解HTTP协议、会写简单脚本、能阅读前端代码 |
| **典型工具** | Python + Requests/Selenium、浏览器自动化(Puppeteer)、AI API |
| **攻击方式** | 自行编写自动化脚本，针对特定漏洞开发绕过逻辑 |
| **投入时间** | 4-8小时（分析+开发+调试） |
| **资金成本** | 0-50元（AI API调用费用，Puter.js/Mistral免费，OCR降级链） |
| **攻击特征** | 有基础的反检测意识，会添加随机延迟和简单行为模拟 |
| **检测概率** | 中（需要服务端行为分析）→ 当前**实际检测概率：极低** |
| **数量规模** | 较多 — 占攻击者总数的15% |

**关键洞察**: 业余技术者能够组合利用多个漏洞（如VULN-001+002+005），实现端到端的自动化流程，包括验证码自动解决。

#### Level 3: 专业攻击者 (Professional Attacker)

| 维度 | 描述 |
|------|------|
| **技术能力** | 高 — 深入理解Web安全、浏览器安全、反检测技术 |
| **典型工具** | 自研工具链、代理池、指纹浏览器、分布式架构 |
| **攻击方式** | 系统性攻击面分析，开发定制化攻击工具，具备完整反检测能力 |
| **投入时间** | 1-3天（完整攻击链开发） |
| **资金成本** | 100-500元（代理、VPS、API费用） |
| **攻击特征** | 行为模式高度拟人，指纹随机化，请求模式分散化 |
| **检测概率** | 低 — 需要高级检测机制 |
| **数量规模** | 少 — 占攻击者总数的4% |

**关键洞察**: 专业攻击者不仅自己刷课，还可能将工具打包为"刷课服务"出售给Level 1用户，形成黑产链条。

#### Level 4: 高级持续性威胁 (APT)

| 维度 | 描述 |
|------|------|
| **技术能力** | 极高 — 国家级/情报级攻击能力，0day储备 |
| **典型工具** | 自研框架、供应链攻击、社工平台、C2基础设施 |
| **攻击方式** | 长期潜伏，多维度渗透，目标不限于刷课（数据窃取、权限提升） |
| **投入时间** | 数周至数月 |
| **资金成本** | 数千至数万元 |
| **攻击特征** | 几乎不可区分于正常用户，使用高级规避技术 |
| **检测概率** | 极低 — 需要专业安全团队和SOC |
| **数量规模** | 极少 — 占攻击者总数不到1% |

**关键洞察**: APT级别攻击者对学习平台的直接兴趣有限，但可能将其作为横向渗透跳板或凭据收集源（学生常用同一密码登录多个系统）。

|----------|---------|---------|---------|---------|

| 供应链攻击 | ❌ | ❌ | ❌ | ✅ |
| 社会工程 | ❌ | ❌ | ⚠️ | ✅ |

| 分布式攻击 | ❌ | ❌ | ✅ | ✅ |
| C2基础设施 | ❌ | ❌ | ❌ | ✅ |

### 2.3 攻击者动机分析

```
攻击动机强度排序:

1. 🔴 时间节约 (最强)     — 课程内容对学生无价值，但学习时间是强制要求
2. 🟠 规避惩罚 (强)       — 未完成课程影响学分/毕业，产生强烈动机
3. 🟡 经济利益 (中)       — 代刷课服务可产生直接经济回报
4. 🟢 技术挑战 (弱)       — 部分攻击者出于技术好奇心
5. ⚪ 恶意破坏 (极弱)     — 极少数攻击者可能意图破坏平台数据完整性
```

**核心洞察**: 攻击者动机极强（涉及学业/毕业），且攻击行为（刷课）在道德灰色地带，导致攻击者群体远大于传统安全威胁场景。这意味着平台面临的攻击规模不是"几个黑客"，而是"大量普通学生"。

## 3. 完整攻击链 (Kill Chain)

本节使用 Cyber Kill Chain 模型，从攻击者视角描述从侦察到目标达成的完整7阶段过程。

### 3.1 阶段一：侦察 (Reconnaissance)

> **攻击者目标**: 了解目标系统的架构、防御机制、攻击面

#### 3.1.1 被动侦察

| 侦察动作 | 方法 | 获取信息 | 难度 |
|----------|------|----------|------|
| 域名信息收集 | WHOIS查询、DNS枚举 | 服务器IP、CDN提供商、子域名 | 极低 |
| 技术栈识别 | HTTP响应头分析、Wappalyzer | Web框架、服务器类型、前端库 | 极低 |
| 公开信息搜索 | 搜索引擎(site:leykeji.com) | 平台功能描述、用户手册、公开文档 | 低 |
| 社交媒体情报 | 贴吧、知乎、QQ群搜索 | 用户反馈、已知漏洞讨论、刷课经验分享 | 低 |

#### 3.1.2 主动侦察

| 侦察动作 | 方法 | 获取信息 | 风险 |
|----------|------|----------|------|
| 前端代码审计 | 浏览器DevTools查看源码 | video.js/online.js完整逻辑、API端点、验证码机制 | 无风险 |
| API端点发现 | Network面板抓包 | 所有API请求格式、参数、响应结构 | 无风险 |
| 验证码样本收集 | 访问/service/code | 验证码复杂度评估、OCR可行性判断 | 无风险 |
| 签名机制分析 | 查看HTML隐藏字段 | appId、nonce、timestamp、sign参数 | 无风险 |
| 防御机制探测 | 尝试异常操作 | 并行检测逻辑、速率限制规则、错误信息 | 极低风险 |

**攻击者发现的关键情报**:

```
情报1: 学习时长API (POST /user/node/study) 仅验证studyTime递增
情报2: 鼠标轨迹收集代码被注释，/service/mouse_log 端点无数据
情报3: 并行检测使用localStorage，键名格式 node_play_{schoolId}{userId}
情报4: 验证码为简单文字图片，/service/code 和 /service/code/aa
情报5: 签名参数在HTML隐藏字段中暴露，/service/sign 返回500
情报6: 页面可见性检测仅在客户端，无服务端验证
情报7: 会话Cookie无CSRF保护，不轮换
情报8: GLM-4V-Flash免费视觉模型可用于验证码识别
```

#### 3.1.3 侦察阶段攻击树

```
目标: 获取足够的系统信息以制定攻击计划
├── [✅ 已完成] 前端源码获取 → 所有防护逻辑可见
├── [✅ 已完成] API格式分析 → 所有端点参数已知
├── [✅ 已完成] 防御机制识别 → 每个防护的绕过方法已确定
└── [✅ 已完成] 攻击成本评估 → 零成本（免费AI模型+浏览器）
```

### 3.2 阶段二：武器化 (Weaponization)

> **攻击者目标**: 基于侦察结果，构建定制化的攻击工具

#### 3.2.1 武器化选择矩阵

#### 3.2.2 武器化关键模块

**模块A: 验证码突破武器 (VULN-002, VULN-005)**

攻击者需要构建一个验证码自动识别管线：

```
验证码突破管线架构:

1. 验证码捕获层
   ├── 拦截验证码图片请求
   ├── 提取图片base64编码
   └── 缓存当前验证码会话

2. AI识别层
   ├── GLM-4V-Flash API调用
   ├── Prompt工程优化
   └── 结果清洗与验证

3. 结果提交层
   ├── 自动填写验证码输入框
   ├── 触发表单提交
   └── 失败重试机制

成本: 零 (Puter.js/Mistral免费，OCR降级链)
成功率: >90% (文字验证码), >75% (点选验证码)
延迟: 2-5秒/次
```

**模块B: 时间伪造武器 (VULN-001)**

攻击者需要构建一个学习时长伪造引擎：

```
学习时长伪造引擎架构:

1. 会话初始化
   ├── POST /user/node/study {studyTime: 1}
   ├── 获取studyId
   └── 建立伪造时间线

2. 定时上报循环
   ├── 计算下一次上报时间 (30秒 + 随机抖动)
   ├── 生成studyTime值 (上次值 + 28秒 + 随机偏差)
   ├── POST /user/node/study {studyTime, studyId, nodeId}
   └── 验证响应（检查need_code状态）

3. 异常处理
   ├── need_code=1 → 调用模块A
   ├── need_code=2 → 调用模块A（图片点选模式）
   ├── 提交失败 → 指数退避重试
   └── 会话过期 → 重新初始化

关键: 服务端仅验证studyTime递增，无真实性校验
```

**模块C: 反检测武器 (VULN-003, VULN-004, VULN-008)**

```
反检测武器架构:

1. localStorage伪造 (VULN-003)
   ├── setInterval(() => {
   │     localStorage.setItem('node_play_' + schoolId + userId, nodeId);
   │   }, 500); // 快于原生的567ms

2. 页面可见性伪造 (VULN-008)
   ├── Object.defineProperty(document, 'visibilityState', {get: () => 'visible'})
   ├── Object.defineProperty(document, 'hidden', {get: () => false})
   └── 拦截visibilitychange事件

3. 鼠标行为模拟 (VULN-004利用)
   ├── 由于sentLog被注释，此模块实际上不需要
   ├── 但攻击者可能仍添加简单模拟以备将来启用
   └── 模拟贝塞尔曲线鼠标轨迹（而非直线移动）

4. 焦点事件伪造
   ├── 拦截window.blur事件
   └── 拦截window.focus事件
```

### 3.3 阶段三：投递 (Delivery)

> **攻击者目标**: 将武器化工具投递到目标系统中执行

#### 3.3.1 投递方式分析

| 投递方式 | 描述 | 隐蔽性 | 部署难度 |
|----------|------|--------|----------|
| **Tampermonkey脚本** | 在浏览器中安装用户脚本 | 低（控制面板可见） | 极低 |
| **浏览器扩展** | 打包为Chrome扩展 | 中（后台运行） | 低 |
| **Bookmarklet** | 书签栏点击执行 | 中（手动触发） | 低 |
| **DevTools控制台** | 直接粘贴代码执行 | 极低（页面关闭失效） | 极低 |
| **独立程序** | Selenium/Puppeteer独立运行 | 高（无浏览器UI） | 中 |

#### 3.3.2 攻击者的投递决策

对于Level 1-2攻击者，**Tampermonkey脚本**是首选投递方式，因为：
- 安装简单，一键导入
- 自动随页面加载运行
- 可跨域调用AI API（GM_xmlhttpRequest）
- 社区分享方便（脚本可分发）

对于Level 3攻击者，**Puppeteer无头浏览器**是更优选择，因为：
- 完全编程控制
- 无可见UI，隐蔽性高
- 可部署在VPS上实现远程刷课
- 可轻松扩展为多账号并行

### 3.4 阶段四：利用 (Exploitation)

> **攻击者目标**: 利用漏洞突破系统防御，执行未授权操作

#### 3.4.1 利用顺序策略

攻击者会按照**从外到内、从简单到复杂**的顺序利用漏洞：

```
利用顺序:

Step 1: 突破登录防线
```

## 4. 单漏洞深度攻击路径

### VULN-001: 学习时长API无服务端真实验证 (CVSS 9.8, Critical)

**攻击前提条件**:
- 已获取有效的登录会话(Cookie)
- 知道目标课程的nodeId

**具体攻击步骤**:

1. **信息收集**: 访问课程页面，从HTML或API响应中提取nodeId列表
2. **会话建立**: 发送 `POST /user/node/study {nodeId, studyTime:1}` 获取studyId
3. **时间伪造**: 以30秒间隔发送递增的studyTime值，不实际播放视频
4. **进度推进**: studyTime按每30秒递增28秒的节奏持续上报
5. **节点完成**: studyTime达到视频总时长后，自动切换到下一个nodeId
6. **批量处理**: 对所有课程所有节点重复步骤2-5

**所需工具/技能**: HTTP客户端(Python requests/fetch)、基本编程能力

**检测难度**: ⭐☆☆☆☆ (极难检测) — 服务端无真实验证逻辑，仅从数据层面无法区分

**影响范围**: 🔴 全平台所有课程 — 所有学习记录均可被伪造

**组合攻击可能性**: **核心锚点漏洞** — 几乎所有组合攻击都以VULN-001为基础

### VULN-002: 登录验证码可被AI自动识别 (CVSS 7.5, High)

**攻击前提条件**:
- 拥有目标平台的账号凭据
- 获取智谱AI API Key（免费注册）

**具体攻击步骤**:

1. **验证码获取**: 访问登录页，请求 `GET /service/code` 获取验证码图片
2. **图片处理**: 将验证码图片转为base64编码
3. **AI识别**: 调用GLM-4V-Flash API，发送图片和识别Prompt
4. **结果清洗**: 移除AI返回中的空格、标点等干扰字符
5. **自动登录**: 将识别结果填入登录表单并提交
6. **失败重试**: 识别失败时刷新验证码重试（最多5次，成功率>95%）

**所需工具/技能**: AI API调用能力、基本图像处理

**检测难度**: ⭐☆☆☆☆ (极难检测) — AI识别行为在服务端看起来与正常人工输入无异

**影响范围**: 🔴 登录入口 — 自动化攻击链的第一道门

**组合攻击可能性**: **入口漏洞** — 与VULN-005组合实现全流程验证码突破

### VULN-003: 并行播放检测仅依赖localStorage (CVSS 8.1, High)

**攻击前提条件**:
- 已登录平台
- 了解localStorage键名格式 `node_play_{schoolId}{userId}`

**具体攻击步骤**:

1. **键名构造**: 从页面HTML中提取schoolId和userId，构造localStorage键名
2. **定时写入**: 在脚本中以500ms间隔（快于原生的567ms）写入当前nodeId
3. **多标签并行**: 打开多个标签页，每个标签页学习不同课程，各自维护localStorage
4. **冲突规避**: 确保同一时刻只有一个标签页写入localStorage（或使用iframe隔离）

**进阶攻击手法**:
- 使用iframe替代多标签页，实现单窗口内的"伪并行"
- 每个iframe独立维护localStorage写入，互相不干扰
- 或采用时分复用策略：多个节点交替上报，共享同一个localStorage键

**所需工具/技能**: JavaScript基础、理解localStorage机制

**检测难度**: ⭐☆☆☆☆ (极难检测) — 服务端无并行学习检测机制

**影响范围**: 🟠 多课程并行 — 攻击效率倍增器

**组合攻击可能性**: 与VULN-009(无速率限制)组合实现多账号并行刷课

### VULN-006: 签名验证端点存在实现错误 (CVSS 7.8, High)

**攻击前提条件**:
- 能访问课程页面（查看HTML源码）

**具体攻击步骤**:

1. **签名参数提取**: 访问任意课程页面，从HTML隐藏字段提取appId、nonce、timestamp、sign
2. **签名参数分析**: 分析签名生成规则（通常为 appId + nonce + timestamp + secretKey 的哈希）
3. **签名伪造**: 使用提取的参数构造任意API请求
4. **重放攻击**: 由于nonce不验证唯一性和timestamp不验证时效，签名可无限复用
5. **端点绕过**: /service/sign 返回500，签名验证流程根本无法完成

**进阶攻击手法**:
- 逆向分析签名算法：如果sign是前端JS生成的，完整算法可从源码获取
- 构造签名生成器：一旦算法已知，可为任意请求生成合法签名
- 批量请求签名：一次提取参数，多次使用

**所需工具/技能**: HTML源码分析、哈希算法理解

**检测难度**: ⭐☆☆☆☆ (极难检测) — 签名参数完全合法，无法区分

**影响范围**: 🟠 认证安全 — 签名机制完全失效

**组合攻击可能性**: 与VULN-010组合可伪造任意认证请求

### VULN-008: 无页面可见性服务端验证 (CVSS 4.8, Medium)

```javascript
document.addEventListener('visibilitychange', e => e.stopImmediatePropagation(), true);
```

**所需工具/技能**: JavaScript、DOM API理解

**检测难度**: ⭐☆☆☆☆ (极难检测) — 服务端无可见性验证通道

**具体攻击步骤**:

1. **会话固定**: 登录前设置目标用户的会话Cookie，登录后Cookie不轮换，攻击者可继续使用旧Cookie
2. **CSRF攻击**: 构造恶意网页，诱骗已登录用户访问，自动提交学习时长请求
3. **会话劫持**: 通过XSS或中间人攻击获取Cookie后，在另一设备上复用
4. **长期潜伏**: 利用会话超时过长的特点，长时间维持窃取的会话

**CSRF攻击概念**:
```html
<!-- 恶意网页中的隐藏表单 -->
<form id="csrf" action="https://scauzj.leykeji.com/user/node/study" method="POST">
    <input type="hidden" name="nodeId" value="目标nodeId" />
    <input type="hidden" name="studyTime" value="9999" />
</form>
<script>document.getElementById('csrf').submit();</script>
```

**所需工具/技能**: Web安全基础、社会工程（CSRF场景）

**检测难度**: ⭐⭐☆☆☆ (低) — CSRF攻击可通过Referer检查检测

**影响范围**: 🟡 会话安全 — 增加会话劫持和CSRF风险

**组合攻击可能性**: 与VULN-006组合可伪造完整的认证链

## 6. 高级攻击手法

> 本节描述**超出已知10个漏洞**的攻击向量。这些攻击不一定需要利用已知漏洞，而是利用系统的架构缺陷和信任模型。

### 6.1 时间侧信道攻击

**攻击原理**: 通过分析API响应时间和数据，推断其他用户的学习进度信息。

**攻击方式**:

1. **学习进度推断**: `/user/node/study` 的响应时间可能因studyTime值不同而有微小差异，攻击者可通过大量采样建立时间模型，推断其他用户的学习完成度
2. **在线状态探测**: `/user/online` 端点的响应可能泄露用户在线状态信息
3. **课程热度分析**: 通过并发测试，推断某门课程的当前在线学习人数
4. **节点完成率推断**: 通过分析studyId的分配模式（如果是递增的），推断平台整体学习进度

**攻击价值**: 信息泄露 — 了解其他用户的学习模式，优化自身刷课策略（如在低峰时段操作）

**检测难度**: ⭐⭐⭐⭐⭐ — 几乎不可能检测，因为请求本身看起来完全正常

### 6.2 竞态条件利用

**攻击原理**: 在并发请求的时间窗口内，利用服务端的检查-执行(check-then-act)间隙。

**攻击方式**:

1. **studyTime回退绕过**: 服务端检查studyTime必须递增。攻击者可在极短时间内（毫秒级）发送两个请求：
   - 请求A: studyTime=100
   - 请求B: studyTime=99（时间回退）
   - 如果请求A和B几乎同时到达，服务端可能在处理A的更新生效前就检查了B，导致回退通过

2. **studyId劫持**: 如果studyId是全局递增的，攻击者可能预测或枚举其他用户的studyId

3. **并行检测绕过**: 利用localStorage写入的时间窗口（567ms间隔），在检测逻辑读取和写入之间的空隙插入伪造数据

**攻击价值**: 可能实现学习时长回退（撤销已学习的时间）或绕过其他递增检查

**检测难度**: ⭐⭐⭐⭐☆ — 需要精密的时间控制，成功率不稳定

### 6.3 供应链攻击: 第三方JS库(CDN)投毒

**攻击原理**: 平台依赖的第三方JavaScript库如果从CDN加载，可能被投毒。

**攻击方式**:

1. **CDN劫持**: 如果平台使用HTTP(非HTTPS)加载第三方JS，中间人攻击者可替换JS内容
2. **CDN供应链污染**: 攻击CDN提供商本身（如npm包投毒），在合法库中注入恶意代码
3. **DNS劫持**: 劫持CDN域名的DNS解析，指向攻击者控制的服务器
4. **子域名接管**: 如果CDN的CNAME指向已失效的云服务，攻击者可接管该子域名

**恶意JS注入后可执行的操作**:
- 窃取所有用户的会话Cookie
- 注入自动刷课代码
- 修改前端验证逻辑
- 窃取用户凭据
- 记录用户键盘输入

**攻击价值**: 🔴 极高 — 可影响平台所有用户

**检测难度**: ⭐⭐⭐☆☆ — 需要SRI(Subresource Integrity)或CSP来防御

### 6.4 社会工程: 针对管理员的钓鱼

**攻击原理**: 不直接攻击技术漏洞，而是通过社会工程攻击平台管理员。

**攻击方式**:

1. **钓鱼邮件**: 伪造"平台升级通知"邮件，引导管理员访问伪造的登录页面
2. **假冒技术支持**: 冒充乐课ji技术支持，以"排查问题"为由获取管理员凭据
3. **水坑攻击**: 在管理员常访问的教育技术论坛植入恶意代码
4. **社工库查询**: 利用泄露的密码数据库，尝试管理员账号的密码重用

**攻击价值**: 获取管理员权限后可修改任意用户的学习记录、查看所有用户数据

**检测难度**: 取决于钓鱼质量，高水准钓鱼几乎不可从技术层面检测

### 6.5 数据泄露: 通过时间上报API推断其他用户信息

**攻击原理**: 学习时长上报API可能泄露可推断其他用户信息的侧信道数据。

**攻击方式**:

1. **studyId序列分析**: 如果studyId是全局递增的，攻击者可通过分析自己获得的studyId来推断：
   - 平台总学习会话数
   - 某时间段内的新增学习会话数
   - 活跃用户数量估算

2. **错误信息泄露**: "提交学时失败【1】"等错误消息可能包含内部状态信息，通过模糊测试不同参数值可能触发更多错误消息

3. **用户ID枚举**: 如果nodeId或其他参数与用户ID存在可预测关系，攻击者可能枚举其他用户的学习记录

4. **时序分析**: 通过在不同时间点发送请求并分析响应时间的统计差异，推断数据库负载（即当前在线学习人数）

**攻击价值**: 🟡 中等 — 主要是信息泄露，不直接威胁学习完整性

**检测难度**: ⭐⭐⭐⭐⭐ — 侧信道攻击从请求层面完全不可检测

### 6.6 指纹伪造: Canvas/WebGL Fingerprint Spoofing

**攻击原理**: 即使平台未来添加浏览器指纹检测，攻击者也能伪造。

**攻击方式**:

1. **Canvas指纹伪造**: 通过Hook Canvas API（toDataURL, toBlob），在原始渲染结果上添加微量噪声，使每次生成的指纹不同
2. **WebGL指纹伪造**: 修改WebGL渲染器信息(RENDERER, VENDOR)，伪造不同的GPU信息
3. **AudioContext指纹伪造**: 修改AudioContext处理结果，伪造不同的音频指纹
4. **字体枚举伪造**: 修改document.fonts API返回的字体列表
5. **浏览器特征伪造**: 修改User-Agent、屏幕分辨率、时区、语言等

**攻击实现概念**:

```
指纹伪造策略:

方式1: 基于Proxy API
  → 拦截所有指纹相关API调用
  → 注入确定性噪声（基于账号ID的哈希）
  → 每个账号产生唯一且稳定的指纹

方式2: 基于指纹浏览器
  → 使用商业指纹浏览器(如Multilogin, GoLogin)
  → 每个账号独立配置指纹参数
  → 成本约5-10元/账号/月

方式3: 基于浏览器扩展
  → 开发自定义扩展拦截指纹API
  → 灵活但需要绕过扩展检测
```

**攻击价值**: 使基于指纹的检测完全失效

**检测难度**: ⭐⭐⭐⭐☆ — 高级指纹检测可发现Proxy痕迹，但难以确定性判断

### 6.7 浏览器扩展攻击

**攻击原理**: 利用学生已安装的浏览器扩展程序的权限来实施攻击。

**攻击方式**:

1. **恶意扩展注入**: 攻击者开发一个看似无害的浏览器扩展（如"课程助手"），实际包含刷课代码
   - 扩展拥有跨域请求权限
   - 可访问所有页面的DOM和Cookie
   - 可在后台持续运行
   - 不受Tampermonkey的限制

2. **合法扩展劫持**: 利用已安装扩展的XSS漏洞
   - 某些翻译/笔记扩展有内容脚本注入漏洞
   - 攻击者可通过构造特制页面触发扩展的XSS
   - 在扩展的权限上下文中执行代码

3. **扩展通信劫持**: 拦截扩展与后台页面的消息通信
   - 窃取扩展存储的敏感数据
   - 伪造扩展消息触发特定行为

**攻击价值**: 🟠 高 — 浏览器扩展权限通常很大，可绕过大多数Web层防御

**检测难度**: ⭐⭐⭐⭐⭐ — 服务端完全无法感知浏览器扩展的行为

传统的修复优先级基于单个漏洞的CVSS评分。但从攻击链视角，应该优先**阻断攻击链的关键节点**，而非修复单个漏洞。

**攻击链关键节点**:

```
攻击链依赖关系:

登录入口 ──→ 学习时长伪造 ──→ 任务完成
```

### 8.4 关键防御措施详细说明

#### P0-1: 服务端验证视频播放真实性 (最关键)

这是**阻断整个攻击链的最重要措施**。具体实现方向：

1. **服务端播放位置追踪**: 服务端维护每个用户的视频播放位置，每次上报studyTime时验证其与播放位置的对应关系
2. **加密播放回调**: 视频播放器定时生成加密的进度令牌（包含时间戳+位置+签名），服务端验证令牌合法性
3. **视频分段验证**: 将视频划分为若干段，每段结束时触发验证，确保视频确实在播放
4. **播放事件序列验证**: 服务端记录play/pause/seek等事件序列，检测异常模式

#### P0-2: 启用鼠标轨迹上传 (最快见效)

这是**2小时即可完成**的修复，效果立竿见影：

1. 取消sentLog函数中的注释，启用数据上传
2. 服务端接收并存储xlogs数据
3. 实现基础的行为模式分析：
   - 检测零鼠标移动（完全自动化特征）
   - 检测过于规律的移动模式
   - 检测异常的移动速度
4. 对异常行为标记告警

#### P1-1: 升级行为式验证码

将简单文字验证码替换为更高级的验证方式：

1. **推荐方案**: 接入商业验证码服务（极验/腾讯验证码）
2. **自建方案**: 滑动拼图验证 + 行为轨迹分析
3. **关键原则**: 验证码的安全性不应依赖"图片有多难识别"，而应依赖"人机行为差异"

| P3完成后 | 放弃攻击 | 放弃攻击 | 放弃攻击(ROI过低) | 阻断99.9%攻击 |

> "放弃攻击"并不意味着攻击不可行，而是攻击成本超过了手动完成课程的成本，攻击者从经济角度选择放弃。
